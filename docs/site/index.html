How it works

Pipelines are directed graphs of agents, tools, and human gates. The engine walks the graph, executes each node, evaluates edge conditions, and routes to the next node. Checkpoints enable resume on failure.

11 key features

📋

Dippin Language

Define pipelines in .dip files — human-readable, version-controlled, lintable. Conditional edges, variable interpolation, per-node models.

🤖

Multi-Provider

Anthropic, OpenAI, and Gemini in one pipeline. Cross-review with three models. Infra-level retries for transient errors.

🎮

TUI Dashboard

Real-time node status, activity streaming, modal human gates. See every LLM turn, tool call, and decision as it happens.

🔀

Parallel Execution

Fan-out to concurrent branches with git worktree isolation. Fan-in joins results. The engine has no special-case parallel code.

🛡

Human-in-the-Loop

Choice, freeform, and hybrid gates. Review modals with glamour-rendered markdown. Split escalation gates for context-specific decisions.

💾

Checkpoint Resume

Every node completion is checkpointed. Resume from where you stopped — the engine replays completed nodes and picks up at the failure point.

🔍

Decision Audit Trail

Every edge selection, condition evaluation, and context snapshot logged to activity.jsonl. Full post-hoc routing reconstruction.

🏗

Embedded Workflows

Three flagship pipelines baked into the binary. tracker build_product works anywhere, for anyone, without cloning the repo.

🧪

Simulation Testing

23 dippin tests verify every edge of build_product.dip. preferred_label, immediately_after, path_contains assertions.

🚀

Autopilot Mode

Replace human gates with LLM judges. Four personas: lax, mid, hard, mentor. Run fully autonomous overnight or in CI.

🔌

Pluggable Backends

Your agents already work great. Switch to claude-code when they need file editing, terminal access, and MCP tools. Per-node or global --backend flag.

Pipeline as code

workflow MyPipeline
  goal: "Build something great"
  start: Begin
  exit: Done

  defaults
    model: claude-sonnet-4-6
    provider: anthropic

  agent Begin
    label: Start

  human AskUser
    label: "What should we build?"
    mode: freeform

  agent Implement
    label: "Build It"
    model: claude-opus-4-6
    prompt:
      The user wants: ${ctx.human_response}
      Implement it following the project's conventions.

  agent Done
    label: Done

  edges
    Begin -> AskUser
    AskUser -> Implement
    Implement -> Done
