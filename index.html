Get running in 60 seconds

# Install
go install github.com/2389-research/tracker/cmd/tracker@latest

# See what's built in
tracker workflows

# Run a pipeline by name — no file needed
tracker build_product

# Supercharge agents with Claude Code
tracker --backend claude-code build_product

# Run fully autonomous overnight
tracker --autopilot mid build_product

# Headless — POST gates to a webhook, resume on callback
tracker --webhook-url https://factory.example.com/gate build_product

# Override workflow params at runtime
tracker --param model=claude-opus-4-7 --param retries=3 build_product

# Cap a run's budget — token, dollar, wall-time ceilings
tracker --max-tokens 100000 --max-cost 500 --max-wall-time 30m build_product

# Check your LLM providers
tracker doctor

How it works

Pipelines are directed graphs of agents, tools, and human gates. The engine walks the graph, executes each node, evaluates edge conditions, and routes to the next node. Checkpoints enable resume on failure.

12 key features

📋

Dippin Language

Define pipelines in .dip files — human-readable, version-controlled, lintable. Conditional edges, variable interpolation, per-node models. Workflow ${params.*} with runtime --param overrides. v0.19.0

🔀

Parallel & Manager Loops

Fan-out to concurrent branches with git worktree isolation, fan-in to join. stack.manager_loop supervises async child pipelines with polling, stop conditions, and mid-execution steering — now declarable natively in .dip via ir.NodeManagerLoop. v0.23.0

📤

Typed Structured Output

Declarative writes: / reads: per node. Declared writes are validated into the pipeline context; reads: pins fidelity for consumed keys. Plus per-node context scoping via node.<id>.<key>. v0.21.0

🤖

Pluggable Backends

Native, claude-code, and acp agent backends. Mix per-node for surgical control, or flip globally with --backend. Same agent.Event stream regardless of backend — your pipeline doesn't change.

🛡

Human-in-the-Loop

Choice, freeform, hybrid, yes/no, and interview gates. Glamour-rendered markdown in review modals. Per-gate timeout and timeout_action to keep unattended runs moving. v0.19.0

🌐

Webhook Gates

Fully headless — --webhook-url POSTs gate prompts to an external service and blocks on a callback. Per-gate HMAC tokens, configurable timeout, outbound Authorization header. Slack bots, email approval, mobile push. v0.17.0

🚀

Autopilot Personas

Replace human gates with LLM judges. Four personas — lax, mid, hard, mentor — for overnight runs and CI. Routes through the claude CLI under --backend claude-code, no API key needed.

💰

Budget Governance

max_total_tokens / max_cost_cents / max_wall_time ceilings via CLI flags, library config, or workflow defaults:. Breach halts the run and fires EventBudgetExceeded. Per-provider cost rollup on Result.Cost. v0.17.0

📒

Library API

Embed tracker in Go code. tracker.Run plus context-aware Diagnose / Audit / Doctor / Simulate returning typed reports. NDJSON event streaming via NewNDJSONWriter. v0.18.0

🧠

Smarter Agents

Episodic memory across retries, plan-before-execute phase, repository localization before the first turn, structured reflection on tool failure, and two-phase verify-after-edit. SWE-bench Lite 59% → 70% baseline. v0.21.0

💾

Checkpoint & Bundle

Every node completion is checkpointed. With WithGitArtifacts, the artifact dir becomes a git repo with per-node commits and checkpoint tags; --export-bundle produces a portable .bundle you can git clone anywhere.

🔍

Diagnose & Audit

Every edge selection, condition evaluation, and outcome logged to activity.jsonl. tracker diagnose classifies deterministic vs. flaky failures; tracker-swebench analyze triages bulk SWE-bench results. v0.22.0

Pipeline as code

workflow MyPipeline
  goal: "Build something great"
  start: Begin
  exit: Done

  defaults
    model: claude-sonnet-4-6
    provider: anthropic

  agent Begin
    label: Start

  human AskUser
    label: "What should we build?"
    mode: freeform

  agent Implement
    label: "Build It"
    model: claude-opus-4-7
    prompt:
      The user wants: ${ctx.human_response}
      Implement it following the project's conventions.

  agent Done
    label: Done

  edges
    Begin -> AskUser
    AskUser -> Implement
    Implement -> Done
